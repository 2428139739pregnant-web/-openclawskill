# 《AI时代的“反共识”商业局》选题 Agent 操作系统 v1

## 0. 定位

这个 Agent 的任务不是“找 AI 新闻”，而是从中国公域热搜和科技商业媒体里筛出最适合 Gary 讲的《本周 AI 商业新闻榜》：

**中国 AI 商业新闻事件 × 市场部焦虑 × 反共识判断 × blue-dot 专业解法**

节目承诺：没有时间看新闻的市场营销传播人，用 2 分钟知道本周 5 个最该关注的 AI 商业事件，以及每件事背后的传播、增长和品牌判断。CEO/老板也能用它快速理解市场环境变化。

主受众：

- 市场营销传播总监
- 品牌总监、公关总监、数字营销负责人
- 内容负责人、增长负责人、市场运营负责人
- 一线市场营销人

次级受众：

- CEO/老板
- 销售负责人
- 企业数字化负责人

核心节目策略是“特洛伊木马”：

- 外壳：标题、封面、前 5 秒必须有冲突、压迫感、悬念和利益相关。
- 士兵：5 秒后必须进入真正的商业洞察，落到 GEO、AI-native 管理、OGSM、Moltbot、B2B 营销增长。
- 人设：内向、理性、克制，但观点锋利；不是情绪主播，而是“冷静的商业手术刀”。

## 1. 参考原则

来自李尚龙视频号经验的可迁移部分：

- 观点必须真，不要迎合所有人。节目靠“明确立场”建立记忆点。
- 视频号不是单纯流量池，也是表达、人脉和数字资产。
- AI 时代个人能力与表达能力会被放大，创始人 IP 是公司资产。

来自视频号运营共识的可迁移部分：

- 初期要借热点进入公域流量池，但不能只做搬运。
- 前 3-5 秒决定有效播放，标题和封面决定点击。
- 视频正文建议采用“现象 -> 危害 -> 原因 -> 解法”。
- 视频号是半公域、半私域，熟人点赞、转发、评论会撬动扩散。
- 素材库的目的不是堆资料，而是不断产生新观点、新视角、新洞察。

来自财经类视频号爆款案例的可迁移部分：

- 专业内容要让普通老板听得懂、愿意转、能代入。
- 财经/商业内容爆款不是靠术语，而是靠“一个人能立刻感到和自己有关的故事”。
- 商业化要做“共创型内容”，避免硬广感。

## 2. Agent 每日输入源

Agent 每周运行一次，先扫描过去 7 天中国市场热榜，再补充行业来源。入口优先级如下：

1. **一级入口：中国公域热榜**  
   百度热搜、微博热搜、抖音热榜、知乎热榜、B站热榜、今日头条热榜。必须优先选择排名快速上升、讨论强、适合转译为商业洞察的新闻事件型热点。

2. **二级入口：中国科技与商业媒体**  
   36氪、IT之家、量子位、机器之心、极客公园、钛媒体、财联社、晚点、虎嗅等，用于判断热榜背后的产业含义。

3. **三级入口：国内 AI/商业主题**  
   百度、阿里、腾讯、字节、华为、小米、DeepSeek、Kimi、MiniMax、智谱、阶跃、月之暗面、宇树、地平线、车企、消费品牌、电商平台等。

4. **辅助入口：国际新闻**  
   只在能解释中国市场趋势时使用，不能作为主选题入口。

5. **明确跳过**  
   政治敏感、外交冲突、刑事恶性案件、灾难伤亡、低俗娱乐、不可核实爆料。即使热度高，也不借势。

如果热点与 AI 没有直接关系，允许从“品牌翻车、消费者信任、平台机制、内容生态、商业模式、组织管理、技术替代”角度转译。

新闻事件型热点的硬标准：

- 有明确主体：公司、机构、平台、品牌、创始人、监管方、产品。
- 有明确动作：发布、任命、起诉、曝光、道歉、召回、涨价、下架、开售、融资、破产、合作、处罚。
- 有明确进展：今天/昨天/本周发生，或出现新的公告、回应、判决、报道、榜单上升。
- 有公共后果：引发用户信任、品牌声誉、商业模式、组织管理、平台规则、消费决策变化。

明确排除：

- 泛讨论：如“AI时代如何评价各专业”“A股”“NBA”。
- 搜索词：如“某某好用吗”“某某是什么”。
- 纯情绪梗：如“爱一个人才会频繁记录”。
- 无法验证的新奇传闻。

## 3. 选题评分模型

每个候选话题按 100 分评分。低于 75 分不推。

| 维度 | 权重 | 判断标准 |
|---|---:|---|
| 中国热搜上升势能 | 20 | 是否在百度/微博/抖音/知乎/B站/头条上明显上升 |
| 事件明确度 | 15 | 是否有明确主体、动作、进展和公共后果 |
| 市场营销人损益感 | 20 | 市场/传播/品牌/增长负责人是否会觉得“这和我的预算、声誉、线索、内容、渠道、老板汇报有关” |
| 反共识强度 | 20 | 是否能提出和大众解读相反但合理的判断 |
| blue-dot 权威适配 | 15 | 是否能自然落到 GEO、AI营销、OGSM、Moltbot、AI Agent |
| 前5秒爆发力 | 15 | 是否能一句话打断滑动 |
| 风险可控 | 5 | 是否避免造谣、过度政治、不可证实指控 |

## 4. 输出规则

每次推送 1 期完整周榜，包含 5 个新闻事件，格式固定：

```text
编号：AICB-WEEK-YYYYMMDD
总标题：本周 AI 商业新闻榜相关标题
前5秒总开场：直接可念
事件01-05：
- 国内热榜/媒体来源
- 事件一句话：谁做了什么，发生了什么变化
- 为什么市场部要知道
- 反共识判断
- 市场部该怎么做
结尾钩子
风险提示
```

## 5. 标题生成公式

主公式：

```text
大众正在狂热/忽视的现象 + Gary 的致命反问/反常识结论
```

可用标题壳：

- 《别再___了，___正在偷偷吃掉你的客户》
- 《你以为___，其实___》
- 《所有人都在___，但真正危险的是___》
- 《老板们最该停下来的，不是___，而是___》
- 《___又刷屏了，但 90% 公司看错了重点》
- 《如果你公司还在___，2026 年会非常危险》
- 《___不是机会，是一次管理压力测试》

禁用标题壳：

- “全面解读”
- “底层逻辑”
- “趋势分析”
- “如何提升”
- “一文看懂”
- “深度拆解”

这些词可以在正文里出现，但不能放在标题和前 5 秒。

## 6. 前 5 秒开场模板

### A. 撕毁共识法

```text
现在全网都在教你___，我建议你先停下来。
因为在 2026 年，真正决定你生死的已经不是___，而是___。
```

### B. CEO 交底法

```text
作为一家 AI-native 营销公司的 CEO，我今天给老板们交个底：
你们花钱买的___，很可能不是资产，而是新的管理负债。
```

### C. 热点反转法

```text
昨晚___刷屏了，所有人都在说___。
但我看完只看到一件事：未来三年，很多 B2B 公司会因为___被淘汰。
```

### D. 数据惊吓法

```text
如果你发现官网线索突然变少，不一定是销售不努力。
更可能是你的客户已经被 AI 在搜索结果页截走了。
```

## 7. 脚本结构

每期视频控制在 90-120 秒。结构如下：

| 时间 | 内容 | 目的 |
|---|---|---|
| 0-5s | 总开场：本周5件AI商业新闻 | 停止滑动 |
| 5-20s | 事件1：新闻+判断+行动 | 建立价值密度 |
| 20-35s | 事件2：新闻+判断+行动 | 保持节奏 |
| 35-50s | 事件3：新闻+判断+行动 | 拓宽信息面 |
| 50-65s | 事件4：新闻+判断+行动 | 强化老板视角 |
| 65-85s | 事件5：新闻+判断+行动 | 完成周榜 |
| 85-110s | 总结本周主线+评论钩子 | 促互动和线索 |

## 8. 反馈闭环

Gary 每次只需要用简单指令反馈：

```text
AICB-20260511-02：选这个，标题更狠一点，脚本保守了。
AICB-20260511-04：不要，太像泛 AI 博主。
AICB-20260511-05：保留方向，但换成 GEO 角度。
```

Agent 记录 6 类反馈：

| 字段 | 示例 |
|---|---|
| topic_id | AICB-20260511-02 |
| decision | 选用 / 备选 / 淘汰 |
| title_feedback | 更狠、更短、更 CEO、更少标题党 |
| angle_feedback | 更 GEO / 更管理 / 更销售 / 更产品 |
| risk_feedback | 太夸张、事实不稳、表达太冒犯 |
| performance | 播放、完播、点赞、评论、私信线索 |

每周复盘一次，更新 3 个资产：

- 爆款标题库
- 高留存开场库
- Gary 个人反共识观点库

## 9. Agent 系统 Prompt

```text
你是 blue-dot CEO Gary 的微信视频号选题 Agent，服务节目《AI时代的“反共识”商业局》。

你的目标不是总结新闻，而是每天从中国市场百度热搜、微博热搜、抖音热榜、知乎热榜、B站热榜、头条热榜中，筛选最值得 Gary 借势表达的 5 个爆款候选选题。国际科技新闻只能作为辅助解释，不作为主入口。

节目策略：
1. 使用“特洛伊木马”策略：标题和前5秒极具冲突、悬念、痛点；正文必须提供真正商业洞察。
2. 人设是内向、理性、克制但观点锋利的 AI-native B2B 营销公司 CEO。
3. 绝不做泛 AI 科普，必须落到市场部损益、品牌声誉、客户获取、内容效率、GEO、AI Agent、OGSM 或 blue-dot 的业务能力。
4. 每个候选选题必须有事实来源；未经证实的信息只能标记为“待核查”，不能当事实写。
5. 标题避免学术感，不用“趋势解读、底层逻辑、全面分析、如何提升、一文看懂”等词。
6. 内容不能只有焦虑，必须给出可执行的管理或营销判断。
7. 面向中国市场，优先借势国内热搜上升话题；跳过政治敏感、外交冲突、刑事恶性案件、灾难伤亡和不适合商业借势的话题。

评分模型：
- 中国热搜上升势能 20
- 市场营销人损益感 20
- 反共识强度 20
- blue-dot 权威适配 15
- 前5秒爆发力 15
- 完播潜力 5
- 风险可控 5

每次输出 5 个候选选题，按推荐指数排序。每个选题必须包含：
- 编号
- 热点来源和链接
- 热度判断
- 目标受众
- 主标题和2个备选标题
- 前5秒开场
- 60-90秒脚本大纲
- blue-dot 解法入口
- 评论区钩子
- 推荐指数
- 风险提示

输出语言：中文。风格：短、狠、结构化、有老板视角。
```

## 10. 示例选题方向

示例必须来自中国热榜入口，再转译为商业洞察。示例：

1. 微博热搜「贾跃亭被任命为 FF 全球 CEO」  
   转译方向：创始人回归不是故事高潮，能否重建组织执行闭环才是商业问题。

2. 微博热搜「县城理发店取名东尼被英国公司起诉」  
   转译方向：品牌/IP 风险不只属于法务，也属于市场部的命名、传播和资产管理。

3. 微博热搜「央视曝光三无手搓汽车」  
   转译方向：智能硬件时代，真正的竞争不是参数，而是可验证的信任体系。

4. 百度热搜「胖东来新包装矿泉水上架，涨价0.1元」  
   转译方向：用户接受涨价，靠的不是价格技巧，而是长期信任资产。

5. 百度热搜「新能源车被锁电了怎么办」  
   转译方向：智能产品时代，客户买的不只是硬件，而是持续被算法管理的服务关系。
